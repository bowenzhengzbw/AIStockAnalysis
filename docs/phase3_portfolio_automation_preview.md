# Phase 3 预览：组合策略与自动化闭环

> 本文聚焦 Phase 3（组合策略、回测分析、Orchestrator 协同与报告自动化）的预览包，旨在在进入开发前确认目标、交付物、技术方案与验收标准，确保最终形成可执行、可复盘的投资闭环。

## 1. 目标概述
- 将 Macro、Policy、Industry、Company、Sentiment、Risk 等 Agent 输出整合为统一信号体系，形成 1-3 年期投资组合建议。
- 建立可重现的回测与绩效分析框架，为策略有效性、风险暴露与归因提供量化依据。
- 实施 Orchestrator（Codex 驱动）协作流，自动协调数据更新、Agent 调用、报告生成与通知。
- 打造投资者界面：调仓建议面板、周月度报告、复盘总结，实现人机共创流程。

## 2. 信号集成与组合引擎
### 2.1 信号标准化
| 信号来源 | 数据实体 | 频率 | 标准化方法 | 备注 |
| --- | --- | --- | --- | --- |
| Macro Sentinel | `macro_view`（周期判断、风格倾向） | 周/月 | Z-score + 阶段标签（扩张/收缩） | 结合风险容忍度映射至行业权重建议 |
| Policy Watcher | `policy_events`（力度、紧迫性、覆盖度） | 事件驱动 | 评分归一化至 0-1，半衰期衰减 | 设立“政策热度指数” |
| Industry Mapper | `industry_outlook`（景气度、催化） | 周/月 | 百分位排名 | 保持行业分类对齐（中信/申万） |
| Company Analyst | `company_scorecard`（财务、估值、信息面） | 季度/事件 | 0-100 分制 | 支持自定义权重覆盖个人偏好 |
| News & Sentiment Scout | `sentiment_alerts`（情绪、舆情热度） | 日/事件 | 情绪分值归一化，设置信号阈值 | 引入黑白名单机制 |
| Risk Controller | `risk_dashboard`（波动、杠杆、集中度） | 日/周 | 风险预算百分比 | 约束组合调整幅度 |

### 2.2 组合构建流程
1. 信号聚合：根据资产、行业、主题维度汇总所有 Agent 评分，生成多层次特征矩阵。
2. 多目标优化：
   - 目标函数：最大化预期收益评分 × 权重，同时最小化波动与行业集中度。
   - 约束条件：单票权重、行业权重、政策敏感度、流动性过滤。
   - 优化工具：`cvxpy`/`pyportfolioopt`，支持可解释的约束配置。
3. 结果输出：
   - 推荐持仓及目标权重。
   - 调仓理由（关联至上游信号）。
   - 期望收益/风险指标、置信度评分。
4. 人工反馈入口：支持投资者对推荐权重进行调整并记录偏好，持续优化。

## 3. 回测与绩效分析
### 3.1 回测框架设计
- **数据准备**：使用 Phase 1-2 落地的数据仓库，构建日度/周度级别的历史信号矩阵。
- **交易规则**：支持多种交易成本设定（千分之几滑点、税费），考虑调仓频率限制。
- **策略类型**：
  - 主策略：综合评分驱动的多因子多行业组合。
  - 策略对照组：仅宏观/政策信号、仅财务指标、被动指数。
- **执行工具**：`backtrader`、`zipline-reloaded` 或自研轻量框架，与 DuckDB/Parquet 数据接口。

### 3.2 绩效输出
- 收益与风险：累计收益、年化收益、波动率、Sharpe、Sortino、最大回撤。
- 因子归因：暴露于成长/价值、大小盘、行业因子；政策事件前后的超额收益分析。
- 时序分析：不同宏观阶段、政策周期下的收益贡献对比。
- 报告模板：自动生成 Markdown/Notebook 报告，附关键图表（收益曲线、行业权重、归因表）。

## 4. Orchestrator 协作与自动化
### 4.1 Orchestrator 职责
- 调度：根据日历触发数据更新、信号计算、回测刷新、报告生成。
- Agent 协同：维护任务队列与上下文，将上游输出写入共享存储并通知下游。
- 审计：记录每次任务执行状态、输入输出摘要、使用的提示词与模型版本。
- 人工接口：提供指令集（如“生成本周调仓建议”“重跑宏观场景”），支持人工中断或重启。

### 4.2 技术实现
- 首选 Codex/自研控制层，结合 `langchain`、`autogen` 等框架执行工具调用。
- 定义标准化的任务协议（JSON Schema），确保 Agent 输出可以被 Orchestrator 消费。
- 引入消息队列或简单事件总线（例如 `Redis Streams`、`SQLite` 任务表），保证顺序与幂等。
- 提供 CLI 与可视化面板查看任务状态、日志、错误重试。

## 5. 报告与界面交付
### 5.1 用户界面原型
- **调仓面板**：展示推荐持仓、调整建议、预期指标、信号溯源；支持导出 Excel/CSV。
- **风险雷达**：实时监控组合风险指标、政策敏感度、行业集中度，提供预警。
- **复盘中心**：历史调仓记录、回测结果、人工反馈汇总。
- **舆情播报**：嵌入 News & Sentiment Scout 摘要，与组合持仓关联。

### 5.2 报告自动化
- 周报：宏观/行业变化 + 组合表现快照 + 关键信号变化。
- 月报：收益归因、风险总结、政策情景复盘、持仓调整建议。
- 事件速递：重大政策/黑天鹅事件触发即时简报。
- 技术实现：`nbconvert` 或 `Jinja2` 模板渲染 Markdown → PDF/HTML，自动推送到邮箱/IM。

## 7. 验收标准
1. **组合引擎**：给定样例信号，能生成满足约束的持仓方案，输出理由透明可追溯。
2. **回测框架**：支持至少 3 年历史回测，指标计算准确，并可快速切换参数。
3. **Orchestrator**：可一键执行“数据更新 → 信号汇总 → 调仓建议 → 报告生成”，异常有可视化日志。
4. **报告/界面**：周/月报模板完整；调仓面板与风险雷达具备可交互或 Demo 水平。
5. **合规记录**：所有自动化流程保留执行日志、使用数据源与模型版本编号。

## 8. 风险与对策
- **信号冲突**：不同 Agent 输出可能方向不一致 → 设定优先级与置信度权重，并提供人工 override 机制。
- **数据延迟**：宏观、财务数据更新滞后 → 引入预测或估算模块，显式标记数据时效。
- **计算资源**：回测与优化可能耗时 → 合理缓存、增量回测，并支持分布式/云端扩展。
- **自动化失误**：Orchestrator 错误造成建议偏差 → 设置多层校验与人工确认环节。
