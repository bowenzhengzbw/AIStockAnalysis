# Module 0.1 — 项目愿景与成功指标（签核稿）

> 本文将 Phase 0 预览中的 Module 0.1 进一步细化为可执行的指标体系与季度目标，用于锁定项目成功标准并指导后续评估工作。

## 1. 使命与范围确认
- **使命**：构建以 AI Agent 为核心的中国 A 股研究闭环，覆盖宏观、政策、行业、公司到组合执行的全过程，支持 1-3 年期投资决策。
- **范围**：在预算 ≤ ¥2,000/年 的数据条件下，实现数据采集、分析、报告自动化与人机协同反馈。
- **核心原则**：信息全面、节奏可控、结论可追溯、人机互补。

## 2. 指标体系
| 维度 | 指标 | 计算方法 | 目标值（首年） | 备注 |
| --- | --- | --- | --- | --- |
| 收益 | 年化超额收益 | 组合收益率 - 基准（沪深 300） | ≥ 6% | 基准可按季度复议 |
| 收益 | 命中率 | 正收益交易数量 / 总交易数量 | ≥ 55% | 以调仓记录为统计口径 |
| 风险 | 最大回撤 | 组合净值峰谷差 | ≤ 15% | 超限触发风控复盘 |
| 风险 | 波动率 | 组合年化波动 | ≤ 基准 * 1.2 | 基准为沪深 300 波动 |
| 效率 | 重大事件响应时间 | 从政策/公告发布到内部事件卡片生成 | < 2 小时 | 工作时间内统计 |
| 效率 | 周报生成时间 | 从触发调度到完成周报初稿 | < 30 分钟 | 依赖自动化程度 |
| 质量 | 数据完整率 | 成功写入的数据条数 / 计划条数 | ≥ 95% | Prefect 日志统计 |
| 质量 | Agent 结论一致性 | 人工评审通过的结论占比 | ≥ 80% | 通过反馈表评估 |

## 3. 季度里程碑
| 季度 | 关键目标 | 评估点 |
| --- | --- | --- |
| Q1 | 数据底座与核心 Agent MVP 上线；建立自动化周报流程 | 数据完整率、事件响应时间达到目标 |
| Q2 | 行业、舆情、风控 Agent 扩展；完善知识库 | Agent 结论一致性 ≥ 70%，知识卡片覆盖重点行业 |
| Q3 | 组合策略与回测上线；启动半自动调仓建议 | 超额收益累计 ≥ 3%，风险指标受控 |
| Q4 | 优化模型、补充另类数据、形成年度复盘机制 | 年化超额收益目标达成；完成年度复盘报告 |

## 4. 基准与比较体系
- **收益基准**：沪深 300 指数（000300.SH），可在半年后评估是否切换为自定义混合基准。
- **风险基准**：沪深 300 波动率与最大回撤。
- **效率基准**：以现有人工流程为对照，目标为自动化后节省 ≥50% 时间。

## 5. 评估流程
1. Prefect + 数据仓库每日记录关键指标，自动汇总至仪表板。
2. 每周由 Orchestrator 自动生成 KPI 摘要，供投资者审阅。
3. 每月召开评估会：对偏离目标的指标制定改进措施。
4. 每季度输出《季度执行复盘》，包含指标趋势、主要贡献/拖累因素、下一季度重点。

## 6. 风险与假设
- **假设**：
  - 数据源在预算内可正常获取；API 限额满足任务频率。
  - 投资者可提供策略假设与估值参数以训练 Agent。
- **风险**：
  - 宏观极端事件导致指标短期失真 → 启用情景分析与临时调整目标。
  - Agent 输出质量不稳 → 引入人工抽检与反馈闭环。
