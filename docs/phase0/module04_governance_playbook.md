# Module 0.4 — 评审机制与治理手册（签核稿）

> 本文定义 Phase 0 及后续阶段的评审节奏、沟通渠道、反馈流程与文档治理规范，确保规划落地与迭代过程透明可控。

## 1. 评审节奏
| 周期 | 内容 | 参与者 | 形式 | 产出 |
| --- | --- | --- | --- | --- |
| 每周例会 | 进度同步、阻塞事项、短期计划 | 投资者、Orchestrator、相关 Agent | 30 分钟线上会议 | 《Weekly Sync》纪要（含行动项） |
| 模块预览评审 | 预览稿展示、确认调整方向 | 投资者、对应模块 Owner | 线上会议或异步文档评审 | 预览稿批注、修订意见列表 |
| 模块签核评审 | 确认模块交付满足验收标准 | 投资者、Owner、相关支持者 | 线上会议 + 记录 | `sign-off` 记录（Git 标签或表单） |
| 月度回顾 | KPI 走势、风险、资源评估 | 投资者、Orchestrator、核心 Agent Owner | 60 分钟会议 | 《Monthly Review》报告、改进行动清单 |
| 季度复盘 | 策略成效、模型迭代路线 | 投资者、全体核心 Agent Owner | 120 分钟会议 | 《Quarterly Retrospective》文档 |

## 2. 沟通与协作渠道
- **即时沟通**：微信/Telegram 群组，主要用于事件通知与快速问答。
- **任务管理**：GitHub Issues + Project Board，追踪每个模块的任务、状态、负责人、截止时间。
- **文档协作**：本仓库 Markdown 为主，结合 Obsidian/Notion 做本地/云端阅读；通过 PR 流程进行版本管理。
- **审批记录**：
  - 模块预览通过：在对应 Issue 添加 `preview-approved` 标签。
  - 模块签核：合并 PR 时在描述中记录版本号与确认人；或在 `docs/phase0/signoff_log.md` 登记。

## 3. 反馈闭环
1. 预览稿提交后，投资者在 48 小时内给出反馈或补充需求。
2. Owner 整理反馈为 TODO 列表，更新 Issue 状态与预计完成时间。
3. 完成调整后重新提交版本（标记 `v0.x` → `v0.x+1`），并在文档开头列出更新摘要。
4. 签核后，任何重大变更需创建新 Issue 并在月度回顾时确认。

## 4. 文档与版本管理
- **目录结构**：
  - `docs/master_plan.md`：总体规划（仅在重大方向调整时更新）。
  - `docs/module_breakdown.md`：模块级别拆解（定期校准）。
  - `docs/phase0/`：Phase 0 模块签核稿（本文件夹）。
  - 后续阶段在 `docs/phaseX/` 目录下维持类似结构。
- **版本标记**：
  - 每份文档头部维护 `版本记录` 表格，记录日期、变更摘要、确认人。
  - Git commit message 遵循 `docs: ...`、`feat: ...` 等约定。
- **模板使用**：提供统一的报告模板（周报、事件卡、行业卡、公司卡、复盘），在 Phase 1/2 实施时补充至 `templates/` 目录。

## 5. 风险与升级路径
| 风险 | 描述 | 升级路径 |
| --- | --- | --- |
| 决策延迟 | 关键反馈超过约定时限 | Orchestrator 提醒 → 月度回顾讨论 → 视情况调整节奏 |
| 信息孤岛 | Agent 之间缺少上下文共享 | 月度回顾纳入“跨 Agent 分享”环节；构建知识库索引 |
| 版本冲突 | 多人同时修改同一文档 | 使用 PR 与 Review 机制；紧急修改后补交变更说明 |
| 需求漂移 | 中途新增需求影响节奏 | 通过 Issues 标记 `scope-change`，并在周会确认优先级 |

## 6. 角色职责速览
| 角色 | 职责 | 关键交付 |
| --- | --- | --- |
| 投资者（Owner） | 明确需求、审批关键节点、提供估值输入 | 签核记录、策略假设 |
| Orchestrator | 任务分配、时间管理、质量把控、汇报整合 | 周/月度报告、任务看板 |
| Data Engineer Agent | 数据接入、调度维护、质量监控 | 数据管道、监控日志 |
| Macro/Policy/Company Agent Owner | 各自模块设计与交付 | 预览稿、报告样例 |
| Supporting Agents | NLP、情绪、风控等专项支持 | 模型服务、分析结果 |
