# 中国 A 股 AI 分析闭环总体规划（预览稿）

> 本文作为项目正式启动前的综合规划说明，旨在帮助项目所有者确认目标范围、关键里程碑以及资源配置。确认后即可进入 Phase 0 的工件制作与评审环节。

## 1. 项目愿景与成功指标
- **目标人群**：拥有深厚估值与交易经验的个人投资者，投资期限 1-3 年。
- **核心愿景**：构建“宏观—政策—行业—公司—组合”全链条的 AI 辅助研究闭环，强调信息全面、响应快速、风控可解释。
- **关键指标**：
  1. 组合绩效：相对沪深 300 或自定义基准实现持续超额收益，并控制最大回撤。
  2. 研究效率：核心报告生成时间缩短 ≥50%，关键事件响应时间 <2 小时。
  3. 透明可追溯：所有 Agent 输出保留来源、提示词、版本记录，支持复盘。

## 2. 数据策略（预算 ≤ ¥2,000/年）
| 分类 | 主数据源 | 预算估计 | 角色映射 | 备注 |
| --- | --- | --- | --- | --- |
| 行情/财务 | Tushare Pro 高级会员 | ~¥1,000 | Macro Sentinel、Company Analyst、Portfolio Strategist | API 友好，适合作为结构化数据主干 |
| 政策/资讯 | 财联社 Lite 或雪球 VIP + 官方政策源 | ~¥1,000（可按需取舍） | Policy Watcher、News & Sentiment Scout | 财联社用于实时快讯，官方源确保权威 |
| 行业/宏观补充 | 国家统计局、央行、发改委公开数据；东方财富数据中心 | 免费 | Macro Sentinel、Industry Mapper | 需自建抓取脚本与校验机制 |
| 舆情/情绪 | 微博热搜、36Kr、雪球社区 | 免费/低成本 | News & Sentiment Scout | 通过爬虫 + LLM 做去噪与分类 |
| 可选扩展 | 企查查（月度开通）、行业资讯 APP | <¥500（按需） | Company Analyst、Industry Mapper | 聚焦特定研究周期，避免全年订阅 |

**数据治理要点**：
- 在 ODS 层统一字段命名、交易日历、复权口径，标记 `source`、`cost_center`、`refresh_cycle`。
- 建立“预算监控”表，统计 API 调用次数与付费周期，提醒续费与超限风险。
- 免费网页数据需加入结构变动监测与失败重试策略，保证长期可用。

## 3. 技术与基础设施蓝图
- **存储层**：本地/云对象存储（S3 兼容）承载原始数据；DuckDB/ClickHouse 作为分析层；Lite 版 Postgres 用于元数据与调度。
- **计算层**：Python（Pandas/Polars）为主，辅以 Notebook + 脚本模式；后续可接入轻量 Spark 或 GPU LLM 推理节点。
- **调度与自动化**：Prefect/Apache Airflow 二选一；日/周/季任务分离，事件驱动任务由 Orchestrator 触发。
- **可视化与界面**：Streamlit/Plotly Dash 构建交互式看板；Markdown + Jupyter 输出详报；长期可考虑 BI（Superset）。
- **安全与合规**：本地密钥保存在 `.env` + 密钥管理工具；记录数据来源、授权协议；日志与访问控制纳入 Phase 0 检查项。

## 4. Agent 体系设计
1. **Macro Sentinel**：聚焦宏观指标、全球联动、政策背景，输出周期判断与风格倾向。
2. **Policy Watcher**：抓取政策文件、公告、行业指导意见；使用 LLM 抽取关键词、影响行业与上市公司清单。
3. **Industry Mapper**：构建行业数据库（产能、价格、渗透率、竞争格局），识别景气变动与催化因素。
4. **Company Analyst**：拉取财务数据、估值、股权结构、公告；结合主人的估值体系输出模型与质地评级。
5. **News & Sentiment Scout**：监控多源资讯，分类为事件类型、情绪标签，形成短评与提醒。
6. **Risk Controller**：跟踪风险因子（波动率、杠杆、政策风险）、执行压力测试、预警组合暴露。
7. **Portfolio Strategist**：综合其他 Agent 信号进行评分、构建候选组合、设定再平衡策略。
8. **Orchestrator（Codex）**：任务分配、数据调度、上下文汇总，控制 Agent 协同流程及审计日志。

**协同流程**（详见 [`agent_system_blueprint.md`](./agent_system_blueprint.md)）：
- Orchestrator 根据日程或触发事件下发任务 → 相应 Agent 调用数据管道 → 生成结构化 JSON + Markdown 摘要 → 写入知识库与看板 → Orchestrator 汇总推送。
- 人类研究者可在任一环节插入备注，系统记录反馈以优化提示词与模型参数。

## 5. 分析框架设计
- **宏观层**：建立领先/同步/滞后指标体系；结合政策力度评估场景（扩张、温和、收缩）；形成季度宏观展望与风格建议。
- **行业层**：定义行业景气度打分（供需、价格、政策、资本开支、竞争格局）；构建行业热力图与催化事件日历。
- **公司层**：
  - 财务质量：盈利能力、成长性、现金流、资本结构四维度打分。
  - 估值：DCF、相对估值、隐含增长率；记录主人自定义假设。
  - 信息面：公告、调研纪要、舆情，与政策事件关联。
- **组合层**：多目标优化（收益、风险、流动性、行业集中度）；设定持仓跟踪与调仓建议；记录执行结果与偏离原因。

## 6. 风险管理与复盘
- 设定风险预算（最大回撤、单票权重、行业集中度），实时监控偏离。
- 构建场景库（政策逆转、宏观下行、黑天鹅事件），定期运行压力测试。
- 建立投后复盘模板：归因分析（宏观 vs. 行业 vs. 个股）、模型表现评估、人机决策差异。

## 7. 知识与资产管理
- 建立结构化数据库（Postgres/ClickHouse）与向量库（FAISS/PGVector）并行：
  - 结构化库存储指标、评分、财务数据。
  - 向量库存储报告、公告摘要、政策文本嵌入。
- 引入版本化笔记系统（例如 Obsidian + Git 或自建 Markdown 仓库），每次决策/观点均有编号与引用。

## 8. 实施路线图（与模块分期对应）
| 阶段 | 目标 | 关键产出 | 预计周期 |
| --- | --- | --- | --- |
| Phase 0 | 签核规划、敲定数据与技术底座 | Charter、数据目录、技术选型、预算监控方案 | 2-4 周 |

## 9. 立即待确认事项
1. 上述数据源与预算方案是否满足预期？是否需要更高频或另类数据？
2. Agent 输出格式与汇报周期：偏好每日速览 + 周度深度，还是按触发事件即时汇报？
3. 技术栈偏好：本地 vs. 云端、现有硬件条件、是否有 GPU 资源。
4. 风险控制重点：是否有特定行业或政策敏感度需要额外监控？
5. 人机分工：哪些环节坚持人工主导、哪些可完全交由 AI 处理？
6. Phase 1 预览文档是否覆盖期望中的数据底座范围与 Agent MVP 功能？如需调整，请在实作前反馈。
7. Phase 2 预览（行业、舆情、风险扩展包）是否满足对行业研究与风控的预期？是否需要新增特定行业或另类数据？
8. Phase 3 预览（组合策略与自动化闭环）是否符合对回测、Orchestrator 与报告自动化的期待？是否有特定组合风格或界面偏好需要提前纳入？

确认以上问题后，即可进入 Phase 0 工件的正式撰写与交付。目前 Phase 0 模块签核稿已完成初版，可在 [`docs/phase0/`](./phase0/) 目录查阅并提出修改意见。
