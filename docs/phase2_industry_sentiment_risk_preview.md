# Phase 2 预览：行业、舆情与风险扩展包

> 本文给出 Phase 2 的预览方案，聚焦 Industry Mapper、News & Sentiment Scout、Risk Controller 三大 Agent 的能力扩充，以及知识库与可视化看板的落地，确保在 Phase 1 数据底座之上形成覆盖宏观-行业-公司-风险的分析闭环。

## 1. 阶段目标
- 建立结构化的行业数据库与景气度模型，支持跨周期比较和政策映射。
- 构建舆情与新闻聚合、分类、情绪打分管线，形成事件级别的提醒与记录。
- 实施组合风险监控与压力测试模块，并与数据仓库、可视化看板联动。
- 完成知识库（结构化 + 向量）与交互式仪表盘 MVP，支撑多 Agent 协同和人工复盘。

## 2. 数据与基础设施扩展
### 2.1 新增数据集
| 层级 | 数据集 | 来源 | 更新频率 | 用途 |

### 2.2 技术组件
- **流式采集**：使用 Prefect/EventBridge 监听快讯和舆情更新，支持 webhook / RSS / 爬虫任务。
- **文本处理**：结合 `spaCy` / `fugashi` + LLM（本地或 API）进行命名实体识别、主题聚类、情绪打分；引入 `keybert` 或 `textrank` 提取关键词。
- **知识库**：
  - 结构化层：Postgres/ClickHouse 表设计 `industry_heatmap`、`sentiment_scores`、`risk_metrics`。
  - 向量层：`chromadb` 或 `pgvector` 存储事件文本嵌入，支持语义检索。
- **可视化**：Streamlit 仪表盘展示行业热力图、事件时间线、风险雷达；结合 Superset/DuckDB 提供 ad-hoc 查询。
- **告警系统**：通过飞书/企业微信 Bot 推送高优先级事件与风险提醒。

## 3. Agent 能力定义
### 3.1 Industry Mapper V2
- **输入**：`industry_stats`、`commodity_prices`、政策事件标签、资金流数据。
- **核心步骤**：
  1. 统一行业分类（申万/中信）并建立映射表，与公司列表对齐。
  2. 计算景气度指标：产能利用率、价格、订单、利润率、资金投入等；形成同比/环比趋势。
  3. 结合政策事件与舆情热度，生成行业催化与风险列表。
  4. 输出行业评分（景气度、政策支持度、资金热度、风险因子）。
- **输出物**：行业热力图（可视化 + JSON）、周/月度行业简报、重点行业 watchlist。

### 3.2 News & Sentiment Scout V2
- **输入**：`news_stream`、`fund_flow_topics`、社交媒体文本。
- **核心步骤**：
  1. 爬虫/接口抓取 → 文本去噪、语言标准化、重复检测。
  2. 事件分类（政策、业绩、供需、公司治理、市场情绪等），并识别涉及的行业/公司。
  3. 情绪打分（-1 到 1）+ 强度评分（曝光度、来源权威度、持续时间）。
  4. 构建事件时间线，识别情绪反转与持续趋势。
- **输出物**：每日情绪快照、事件卡片、周度情绪回顾、异常波动告警。

### 3.3 Risk Controller V2
- **输入**：`risk_factors`、行业/公司评分、宏观场景、组合持仓假设。
- **核心步骤**：
  1. 计算组合层风险：VaR、CVaR、最大回撤、Beta、行业集中度。
  2. 构建压力测试：政策逆转、需求收缩、成本上升、资金流出等情景，量化影响。
  3. 监控风险指标与阈值：触发再平衡建议或减仓提示。
  4. 生成风险复盘报告，记录预警触发与处理结果。
- **输出物**：风险仪表盘、预警列表、压力测试报告、调仓建议草案。

## 4. 任务拆解与排期（建议 8-12 周）
| 周次 | 任务 | 交付物 | 依赖 |
| --- | --- | --- | --- |
| 1-2 | 行业分类与数据源落地 | 行业映射表、采集脚本、样例数据 | Phase 1 数据仓库稳定 |
| 3-4 | 行业景气模型迭代 | 景气度指标计算、热力图原型 | 行业数据初步可用 |
| 5-6 | 舆情采集与情绪模型 | 爬虫/接口脚本、情绪打分流程、事件卡片模板 | 新闻源接入授权 |
| 7-8 | 风险因子库与压力测试框架 | 因子计算脚本、情景分析模型、风险仪表盘草稿 | 行情/财务数据完整 |
| 9 | 可视化与知识库集成 | Streamlit/Superset 看板、语义检索接口 | 上述模块可用 |
| 10-11 | Agent 协同与告警机制 | 自动化报告、推送工作流、集成测试 | Orchestrator 基础能力 |
| 12 | 评审与回顾 | Phase 2 交付清单、复盘报告、优化建议 | 全部任务完成 |

## 5. 验收标准
1. **数据完整性**：行业/舆情/风险数据在样例周期内覆盖率 ≥ 90%，自动重试机制有效。
2. **分析质量**：
   - Industry Mapper 报告体现数据 + 政策 + 资金三重维度，评分逻辑透明。
   - News & Sentiment Scout 情绪打分与人工校验一致性 ≥ 70%，重要事件推送延迟 < 2 小时。
   - Risk Controller 能输出至少 5 个压力测试情景，并提供定量影响评估。
3. **系统可用性**：Streamlit 看板响应 < 3 秒；知识库检索命中率（Top3）≥ 80%。
4. **协同与审计**：每个 Agent 输出具备版本号、数据来源引用、提示词记录；告警日志可追溯。

## 6. 风险与缓解措施
- **数据授权变动**：提前准备备用源（Wind 移动版、东方财富数据中心）；关键指标支持手动导入。
- **舆情噪声**：引入规则过滤（黑名单、来源权重），并通过人工反馈迭代情绪模型。
- **模型复杂度**：分阶段上线（先规则后模型），避免一次性投入过多训练工作。
- **资源消耗**：文本处理与向量库可能占用存储与计算，需评估本地硬件并考虑增购 SSD/GPU。

## 7. 下一步沟通点
- 确认行业分类标准与重点关注行业（如先进制造、科技、消费等）。
- 明确舆情源优先级与告警阈值，避免信息过载。
- 讨论风险指标与压力场景是否需要根据投资风格（成长/价值/高分红）调整权重。
- 确定 Streamlit 看板与知识库的使用场景（桌面端、本地/云部署）。

确认本预览方案后，可进入 Phase 2 的详细设计与开发。
