# Phase 0 项目启动章程（待确认）

> 本章程用于在正式进入编码之前，对项目范围、交付物、角色分工以及评审机制达成一致。通过本章程确认后，即可启动 Phase 0 的具体文档与原型制作工作。

## 1. 项目定位与核心假设
- **目标用户**：具备深厚估值能力的个人投资者，关注中国 A 股 1-3 年投资机会。
- **投资哲学**：自上而下（宏观/政策）与自下而上（行业/公司）并重，强调事件驱动与政策敏感度。
- **AI 协同原则**：AI Agent 负责信息采集、初步分析与情景推演，人类负责关键假设、估值输入与最终决策。

## 5. 评审与反馈流程
1. 每个模块先提交 **预览稿**（Markdown + 可选图表），投资者给出意见。
2. 根据反馈迭代至 **签核版本**，标记版本号（如 v0.1 → v0.2）。
3. 所有签核稿统一归档到 `docs/phase0/` 子目录（Phase 0 实施时创建）。
4. 使用 Issue/任务列表跟踪未解决项，避免遗漏。

## 6. 风险与应对
| 风险 | 描述 | 应对措施 |
| --- | --- | --- |
| 数据权限延迟 | 付费接口申请或验证码机制导致接入推迟 | 提前准备所需证件；设置临时替代数据源 |
| 技术方案不匹配 | 现有硬件不足以支持计划中的计算或存储 | 提供轻量化备选方案（如本地 DuckDB + 分批处理） |
| 需求变更频繁 | 在 Phase 0 期间出现额外需求，影响时间表 | 通过评审机制模块建立变更流程和优先级决策 |
